## 📖 Шаг 1 — Зачем нужны условия

Программа без условий — прямая линия: шаг 1, шаг 2, шаг 3, конец. Условие даёт программе **развилку**: если это правда — поворачиваем налево, иначе — направо.

Аналогия: ты едешь по Берлину. Навигатор говорит — если пробка, поворачивай здесь, иначе едь прямо. Твоя программа делает то же самое.

## 📖 Шаг 2 — if

Простейшая форма: "если условие истинно — выполни код":

```python
temperature = 92

if temperature > 90:
    print("Перегрев! Снижаю нагрузку.")
```

Вывод:
```
Перегрев! Снижаю нагрузку.
```

Поставь `temperature = 80` — ничего не выведется. Условие ложно, блок пропускается.

**Структура:**
```python
if условие:
    код_строка_1   # 4 пробела — обязательно!
    код_строка_2
    код_строка_3
# дальше программа продолжается в любом случае
```

⚠️ Отступы — это синтаксис, не украшение
> Python определяет блоки **через отступы**. Без отступа — `IndentationError` и программа не запустится вообще.
> ```python
> if temperature > 90:
> print("Ошибка!")       # IndentationError!
>
> if temperature > 90:
>     print("Правильно") # ✓ 4 пробела
> ```

## 📖 Шаг 3 — if / else

Два варианта: "если истинно → A, иначе → B":

```python
temperature = 74

if temperature > 90:
    print("Перегрев!")
else:
    print("Температура в норме.")
```

Вывод:
```
Температура в норме.
```

Один из двух блоков **всегда** выполняется — никогда оба, никогда ни один.

**Ещё примеры:**

```python
speed_rpm = 1200

if speed_rpm >= 2000:
    print("Охлаждение активно")
else:
    print("Вентилятор почти не работает — проверь систему")

# ---

city = input("Введи город: ")

if city.lower() == "berlin":
    print("Столица Германии — 3.6 млн жителей")
else:
    print(f"{city} — не Берлин")

# ---

ram_gb = 8

if ram_gb >= 16:
    status = "достаточно"
else:
    status = "маловато — закрой лишние приложения"

print(f"RAM {ram_gb} GB — {status}")
# → RAM 8 GB — маловато — закрой лишние приложения
```

## 📖 Шаг 4 — if / elif / else

Когда вариантов больше двух — используй `elif` (сокращение от "else if"):

```python
temperature = 83

if temperature >= 95:
    print("🔴 КРИТИЧНО — немедленно закрой программы!")
elif temperature >= 80:
    print("🟡 ВНИМАНИЕ — температура высокая")
elif temperature >= 60:
    print("🟢 НОРМА — всё хорошо")
else:
    print("🔵 ХОЛОДНО — нагрузка минимальна")
```

Вывод:
```
🟡 ВНИМАНИЕ — температура высокая
```

**Как Python проходит цепочку:**

1. `83 >= 95` → False — пропустить
2. `83 >= 80` → True — **выполнить этот блок и выйти**
3. Остальные `elif` и `else` **не проверяются**

⚠️ Порядок условий важен — всегда от строгого к мягкому
> ```python
> # ❌ НЕПРАВИЛЬНО — условие >= 60 поглотит всё
> if temperature >= 60:    # при 83 тоже True → всегда здесь
>     print("норма")
> elif temperature >= 80:
>     print("внимание")    # никогда не достигнет!
>
> # ✓ ПРАВИЛЬНО — сначала самое строгое
> if temperature >= 95:
>     print("критично")
> elif temperature >= 80:
>     print("внимание")
> elif temperature >= 60:
>     print("норма")
> ```

## 📖 Шаг 5 — Много elif

Количество `elif` не ограничено — сколько нужно вариантов, столько пишешь:

```python
cpu_cores = int(input("Количество ядер CPU: "))

if cpu_cores >= 16:
    tier = "Рабочая станция — флагман"

    tier = "High-end — топовый игровой"

    tier = "Entry-level — базовый"

    tier = "Старое железо"
else:
    tier = "Однопоточный — очень старый"

```

Пример работы (ввод 10):
```
10 ядер → Mid-range — хороший игровой
```

## 📖 Шаг 6 — Вложенные условия

Условие внутри условия — для многоуровневых проверок:

```python
ram_pct  = 88
cpu_temp = 91

if cpu_temp > 90:
    print("CPU перегрет!")
    if ram_pct > 85:
        print("И RAM почти полная — критическая ситуация!")
    else:
        print("Но RAM ещё держится.")
else:
    if ram_pct > 85:
        print("CPU в норме, но RAM переполнена")
    else:
        print("Система работает стабильно")
```

Вывод:
```
CPU перегрет!
И RAM почти полная — критическая ситуация!
```

💡 Вложение больше 2 уровней — сигнал переписать
> Глубокое вложение трудно читать. В следующем уроке узнаешь как заменять его на `and`/`or`.

## 📖 Шаг 7 — Тернарный оператор

Когда нужно присвоить одно из двух значений — запишешь в одну строку:

```python
# Длинная версия
temperature = 85
if temperature > 80:
    label = "горячо"
else:
    label = "норма"

# Тернарный оператор — то же самое
label = "горячо" if temperature > 80 else "норма"
print(label)   # → горячо
```

**Синтаксис:** `значение_A if условие else значение_B`

```python
ram_gb   = 32
result   = "достаточно" if ram_gb >= 16 else "мало"
print(result)   # → достаточно

is_on    = True
icon     = "🟢" if is_on else "🔴"
print(icon)     # → 🟢

score    = 1850
rank     = "gold" if score >= 2000 else "silver" if score >= 1500 else "bronze"
print(rank)     # → silver
```

## 📖 Шаг 8 — Что Python считает True и False

Python оценивает **любое значение** как `True` или `False`:

```python
# Считается False — "пустые" значения
print(bool(0))        # → False
print(bool(0.0))      # → False
print(bool(""))       # → False  (пустая строка)
print(bool(None))     # → False
print(bool([]))       # → False  (пустой список)

# Считается True — всё остальное
print(bool(1))          # → True
print(bool(-5))         # → True
print(bool("Berlin"))   # → True
print(bool(42))         # → True
```

**Практическое применение:**

```python
# Проверка что список не пустой
processes = ["chrome.exe", "discord.exe"]
if processes:
    print(f"Запущено: {len(processes)} процессов")

# Проверка что строка не пустая
name = input("Имя компьютера: ")
if name:
    print(f"Компьютер: {name}")
else:
    print("Имя не введено")

# Проверка числа
free_gb = 0
if not free_gb:
    print("Нет свободного места!")
```

## 📖 Шаг 9 — pass — пустой блок

Когда нужна заглушка — блок который пока ничего не делает:

```python
temperature = 72

if temperature > 90:
    pass   # TODO: добавить логику аварийного охлаждения
else:
    print("Норма")
```

Без `pass` Python выдаст `IndentationError` — пустые блоки недопустимы.

## 🔨 Практика — PC Booster

```python
print("=== PC Booster — Диагностика ===")
print()

cpu_temp = float(input("Температура CPU (°C): "))

for label, value, thresholds in [
    ("CPU",  cpu_temp, (95, 80)),
    ("RAM",  ram_pct,  (95, 80)),
    ("Диск", disk_pct, (95, 80)),
]:
    crit, warn = thresholds

        status = "🟡 ВНИМАНИЕ"
    else:
        status = "🟢 НОРМА"
    print(f"  {label:<5}: {value:>5.1f}  — {status}")
```

## Задание

PC Booster должен определять состояние системы по температуре CPU. Напиши программу которая запрашивает температуру и выводит статус.

**Дано:**
- Пользователь вводит температуру CPU (°C)

**Нужно:**
- Если >= 95 → вывести `"🔴 КРИТИЧНО — немедленно закрой программы!"`
- Если >= 80 → вывести `"🟡 ВНИМАНИЕ — температура высокая"`
- Если >= 60 → вывести `"🟢 НОРМА — всё хорошо"`
- Иначе → вывести `"🔵 ХОЛОДНО — нагрузка минимальна"`

**Ожидаемый вывод:**
```
Температура CPU (°C): 83
🟡 ВНИМАНИЕ — температура высокая
```

💡 Подсказка 1
> Используй цепочку `if / elif / else`. Начни с самого строгого условия (>= 95).

💡 Подсказка 2
> Порядок важен! Если сначала проверить `>= 60`, то при температуре 83 сработает именно оно, а `>= 80` не дойдёт.

💡 Подсказка 3
> Ввод через `float(input(...))` — температура может быть дробной.

## Решение

```python
temp = float(input("Температура CPU (°C): "))

if temp >= 95:
    print("🔴 КРИТИЧНО — немедленно закрой программы!")
elif temp >= 80:
    print("🟡 ВНИМАНИЕ — температура высокая")
elif temp >= 60:
    print("🟢 НОРМА — всё хорошо")
else:
    print("🔵 ХОЛОДНО — нагрузка минимальна")
```

**Разбор:**
1. `float(input(...))` → получаем температуру как дробное число
2. Python проверяет условия **сверху вниз** и останавливается на первом `True`
3. При `temp = 83`: `83 >= 95` → False, `83 >= 80` → **True** → выполняет этот блок
4. Остальные `elif` и `else` **не проверяются** — Python уже вышел из цепочки
5. Если бы порядок был `>= 60` первым — при 83 сработал бы он (ошибка!)

## ❌ Частые ошибки

🐛 `=` вместо `==` в условии
> ```python
> if temp = 85:    # SyntaxError — это присваивание, не сравнение
> if temp == 85:   # ✓ правильно
> ```

🐛 Неправильный порядок elif — от мягкого к строгому
> ```python
> # ❌ Условие >= 60 поглотит всё включая 80 и 95
> if temp >= 60: print("норма")
> elif temp >= 80: print("внимание")  # недостижимо!
> ```

🐛 Нет отступа
> ```python
> if temp > 80:
> print("Горячо")    # IndentationError!
>
> if temp > 80:
>     print("Горячо")  # ✓
> ```

## 💡 Заметки на полях

💡 elif vs несколько if — разница важна
> ```python
> # Несколько if — все проверяются независимо
> if temp > 95: print("критично")
> if temp > 80: print("внимание")   # тоже сработает при temp=96!
>
> # elif — останавливается на первом True
> if temp > 95: print("критично")
> elif temp > 80: print("внимание") # не сработает при temp=96
> ```
