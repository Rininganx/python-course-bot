В прошлом уроке ты писал условия вроде `temperature > 90`. Это сравнение — оно возвращает `True` или `False`. Но что если нужно проверить сразу два условия?

"Температура высокая **И** RAM почти полная" — вот тут нужны булевы операторы. Они позволяют объединять несколько сравнений в одно условие.

## 📖 Шаг 2 — Операторы сравнения

Все операторы сравнения возвращают `True` или `False`:

```python
a = 10
b = 20

print(a == b)    # → False  (равно?)
print(a != b)    # → True   (не равно?)
print(a > b)     # → False  (больше?)
print(a < b)     # → True   (меньше?)
print(a >= b)    # → False  (больше или равно?)
print(a <= b)    # → True   (меньше или равно?)
```

### Полная таблица

Оператор • Значение • Пример • Результат
`==` • Равно • `5 == 5` • `True`
`!=` • Не равно • `5 != 3` • `True`
`>` • Больше • `10 > 5` • `True`
`<` • Меньше • `3 < 8` • `True`
`>=` • Больше или равно • `10 >= 10` • `True`
`<=` • Меньше или равно • `7 <= 10` • `True`

## 📖 Шаг 3 — Сравнение строк

```python
city = "Munich"

print(city == "Munich")     # → True
print(city == "munich")     # → False  (регистр важен!)
print(city != "Berlin")     # → True

# Правильное сравнение без учёта регистра
user_input = "MUNICH"
print(user_input.lower() == "munich")   # → True

# Строки сравниваются лексикографически (как в словаре)
print("Amsterdam" < "Berlin")    # → True  (A раньше B)
print("Vienna" > "Prague")       # → True  (V позже P)
```

## 📖 Шаг 4 — Операторы in и is

```python
# in — проверка вхождения
filename = "setup_berlin_v2.exe"

print(".exe" in filename)         # → True
print(".msi" in filename)         # → False
print("berlin" in filename)       # → True
print(".exe" not in filename)     # → False

# Практика — проверка расширения
allowed_ext = [".exe", ".msi", ".bat"]
ext = ".exe"
print(ext in allowed_ext)         # → True

# is — проверка идентичности (используй только для None)
result = None

if result is None:
    print("Данные ещё не получены")

if result is not None:
    print("Данные есть")
```

💡 `is` только для None, True, False
> `==` сравнивает **значения** — `5 == 5.0` → `True`.
> `is` сравнивает **идентичность объекта в памяти**. Для обычных чисел и строк — используй `==`. Для `None` — используй `is`.

## 📖 Шаг 5 — and (и)

`and` возвращает `True` только если **оба** условия True:

```python
cpu_temp = 88
ram_pct  = 92

print(cpu_temp > 80 and ram_pct > 80)   # → True  (88>80 и 92>80 — оба True)
print(cpu_temp > 80 and ram_pct > 95)   # → False (92>95 — False, значит False)
print(cpu_temp > 95 and ram_pct > 80)   # → False (88>95 — False)
print(cpu_temp > 95 and ram_pct > 95)   # → False (оба False)
```

**Таблица истинности `and`:**

A • B • A and B
True • True • **True**
True • False • False
False • True • False
False • False • False

Запомни: `and` — строгий. Нужно чтобы **все** были True.

────────────────────
## 📖 Шаг 6 — or (или)

`or` возвращает `True` если **хотя бы одно** условие True:

```python
cpu_temp = 88
ram_pct  = 60

print(cpu_temp > 80 or ram_pct > 80)    # → True  (88>80 — True, хватит)
print(cpu_temp > 95 or ram_pct > 50)    # → True  (60>50 — True)
print(cpu_temp > 95 or ram_pct > 80)    # → False (оба False)
```

**Таблица истинности `or`:**

A • B • A or B
True • True • True
True • False • **True**
False • True • **True**
False • False • False

Запомни: `or` — мягкий. Достаточно **одного** True.

────────────────────
## 📖 Шаг 7 — not (не)

`not` инвертирует значение:

```python
boosted = False

print(not boosted)         # → True
print(not True)            # → False
print(not False)           # → True

# Практика
is_running = True
print(not is_running)      # → False

temp_ok = True
if not temp_ok:
    print("Температура вышла за норму!")
else:
    print("Температура в порядке")
# → Температура в порядке
```

## 📖 Шаг 8 — Короткое замыкание

Python умный: он не проверяет второе условие если первое уже решает исход:

```python
# and — если первое False, второе не проверяется вообще
x = 0
if x != 0 and 100 / x > 5:   # 100/0 не выполнится — x != 0 уже False
    print("ok")
else:
    print("x равен нулю, деление пропущено")
# → x равен нулю, деление пропущено

# or — если первое True, второе не проверяется
name = "Lena"
if name or len(name) > 100:   # второе не проверяется — name уже True
    print("Имя есть")
# → Имя есть
```

💡 Короткое замыкание защищает от ошибок
> Всегда ставь "защитное" условие первым:
> ```python
> if value != 0 and 100 / value > 5:   # безопасно
> if 100 / value > 5 and value != 0:   # опасно — деление выполнится первым
> ```

## 📖 Шаг 9 — Приоритет операторов

Python вычисляет в таком порядке (от высшего к низшему):

1. Сравнения: `>`, `<`, `>=`, `<=`, `==`, `!=`, `in`, `is`
2. `not`
3. `and`
4. `or`

```python
# and раньше or — может удивить
result = True or False and False
# Сначала: False and False = False
# Потом:   True or False  = True
print(result)   # → True

# Со скобками — явно и понятно
result = (True or False) and False
# Сначала: True or False = True
# Потом:   True and False = False
print(result)   # → False
```

💡 Всегда используй скобки при смешивании and и or
> ```python
> # Без скобок — непонятно что имелось в виду
> if temp > 90 or ram > 90 and disk > 80:
>
> # Со скобками — явно
> if temp > 90 or (ram > 90 and disk > 80):
> ```

## 📖 Шаг 10 — Цепочки сравнений

Python позволяет записывать диапазоны компактно:

```python
temp = 75

# Обычный способ
if temp >= 60 and temp < 80:
    print("Норма")

# Питоновский способ — цепочка
if 60 <= temp < 80:
    print("Норма")

# Больше примеров
ram_pct = 67
if 0 <= ram_pct <= 100:
    print("Значение корректно")

score = 1850
if 1500 <= score < 2000:
    print("Silver tier")
```

## Задание

PC Booster проверяет готовность системы к игре. Напиши программу которая получает данные и выдаёт вердикт.

**Дано:**
- Пользователь вводит: RAM (GB), температуру CPU (°C), свободное место на диске (GB)

**Нужно:**
1. Проверить каждое условие отдельно и сохранить результат в переменную:
   - `ram_ok` = RAM >= 16
   - `temp_ok` = CPU < 80
   - `disk_ok` = свободное место >= 50
2. Вывести каждое условие с галочкой ✗ или ✓
3. Если **все три** True → "Готов к игре!", иначе — "Система не готова"

**Ожидаемый вывод:**
```
RAM:   ✓ 32 GB
CPU:   ✗ 85°C
Диск:  ✓ 120 GB

Система не готова — реши проблемы выше
```

💡 Подсказка 1
> Сохрани результат каждого сравнения в переменную: `ram_ok = ram_gb >= 16`

💡 Подсказка 2
> Для вывода ✓/✗ используй тернарный оператор: `f"{'✓' if ram_ok else '✗'} {ram_gb} GB"`

💡 Подсказка 3
> Объедини условия через `and`: `ready = ram_ok and temp_ok and disk_ok`

## Решение

```python
ram_gb  = int(input("RAM (GB): "))
cpu_temp = float(input("CPU температура (°C): "))
free_disk = int(input("Свободно на диске (GB): "))

ram_ok  = ram_gb >= 16
temp_ok = cpu_temp < 80
disk_ok = free_disk >= 50

print()
print(f"RAM:   {'✓' if ram_ok else '✗'} {ram_gb} GB")
print(f"CPU:   {'✓' if temp_ok else '✗'} {cpu_temp}°C")
print(f"Диск:  {'✓' if disk_ok else '✗'} {free_disk} GB")
print()

ready = ram_ok and temp_ok and disk_ok
print("Готов к игре!" if ready else "Система не готова — реши проблемы выше")
```

**Разбор:**
1. `ram_ok = ram_gb >= 16` → сравнение возвращает `True` или `False`, сохраняем в переменную
2. `and` объединяет условия: `ready = True and True and True` → `True`
3. Если хотя бы одно `False` → `ready = False` (строгий `and`)
4. `'✓' if ram_ok else '✗'` → тернарный оператор выбирает символ для вывода
5. `"Готов!" if ready else "Не готова"` → финальный вердикт одной строкой

## ❌ Частые ошибки

🐛 `=` вместо `==`
> ```python
> if name = "Lena":    # SyntaxError
> if name == "Lena":   # ✓
> ```

🐛 Сравнение строк без нормализации регистра
> ```python
> status = "Running"
> if status == "running":          # False! Регистр разный
> if status.lower() == "running":  # ✓
> ```

🐛 Использование == для None
> ```python
> if x == None:   # работает, но плохой стиль
> if x is None:   # ✓ правильно
> ```

🐛 Забыть скобки при смешивании and/or
> ```python
> # Неожиданный результат из-за приоритета
> if a or b and c:           # сначала b and c, потом or a
> if a or (b and c):         # явно и понятно
> ```
