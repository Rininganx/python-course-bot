## 📖 Шаг 1 — for vs while

В прошлом уроке ты изучил `for` — он идеален когда знаешь заранее сколько повторений нужно.

`while` — это другая история. Он повторяет код **пока условие истинно**. Не 5 раз, не 100 раз — а до тех пор пока что-то не изменится.

Ситуация • Используй
Знаешь сколько раз повторить • `for`
Повторяешь пока условие выполнено • `while`
Ждёшь правильного ввода от пользователя • `while`
Мониторинг в реальном времени • `while`
Игровой цикл (пока не вышел из игры) • `while`
Перебираешь элементы списка • `for`

## 📖 Шаг 2 — Базовый синтаксис

```python
while условие:
    # тело цикла — выполняется пока условие True
    код
# здесь цикл завершён
```

Простой пример — вывести числа от 1 до 5:

```python
count = 1

while count <= 5:
    print(count)
    count += 1
```

Вывод:
```
1
2
3
4
5
```

**Что происходит на каждом шаге:**

Шаг • count • count <= 5 • Действие
1 • 1 • True • вывести 1, count = 2
2 • 2 • True • вывести 2, count = 3
3 • 3 • True • вывести 3, count = 4
4 • 4 • True • вывести 4, count = 5
5 • 5 • True • вывести 5, count = 6
6 • 6 • False • выйти из цикла

## 📖 Шаг 3 — Бесконечный цикл

`while True` — цикл который никогда не завершается сам. Выходят из него через `break` (следующий урок) или закрытием программы (`Ctrl+C`):

```python
while True:
    command = input("Команда (exit для выхода): ")

    if command == "exit":
        print("Выход...")
        break   # прерываем цикл

    print(f"Выполняю: {command}")
```

Пример работы:
```
Команда (exit для выхода): scan
Выполняю: scan
Команда (exit для выхода): boost
Выполняю: boost
Команда (exit для выхода): exit
Выход...
```

⚠️ Бесконечный цикл без break — программа зависнет
> Если забыть условие выхода — программа будет работать вечно.
> `Ctrl+C` в терминале — принудительная остановка.

## 📖 Шаг 4 — Валидация ввода

Одно из главных применений `while` — просить ввод снова и снова пока не получишь правильный:

```python
# Пример 1 — проверка диапазона
while True:
    temp = float(input("Введи температуру CPU (-50 до 150°C): "))
    if -50 <= temp <= 150:
        break
    print(f"Ошибка: {temp}°C вне допустимого диапазона. Попробуй снова.")

print(f"Принято: {temp}°C")
```

```

Ошибка: 999.0°C вне допустимого диапазона. Попробуй снова.
Введи температуру CPU (-50 до 150°C): -200
Ошибка: -200.0°C вне допустимого диапазона. Попробуй снова.
Введи температуру CPU (-50 до 150°C): 75
Принято: 75.0°C
```

valid_profiles = ["game", "work", "silent"]

## 📖 Шаг 5 — Счётчик с условием

```python
# Попытки входа — максимум 3
attempts = 0
max_attempts = 3
correct_pin = "1234"

while attempts < max_attempts:
    pin = input("Введи PIN: ")
    attempts += 1

    if pin == correct_pin:
        print("✓ Доступ разрешён!")
        break
    else:
        remaining = max_attempts - attempts
        if remaining > 0:
            print(f"✗ Неверный PIN. Осталось попыток: {remaining}")
        else:
            print("✗ Превышено количество попыток. Блокировка.")
```

Пример работы:
```
Введи PIN: 9999
✗ Неверный PIN. Осталось попыток: 2
Введи PIN: 0000
✗ Неверный PIN. Осталось попыток: 1
Введи PIN: 1234
✓ Доступ разрешён!
```

## 📖 Шаг 6 — Накопление в while

Паттерн "собирать данные пока не введут стоп":

```python
print("Введи процессы для мониторинга (пустая строка — стоп):")

monitored = []
total_ram = 0

while True:
    name = input("Процесс (Enter для завершения): ").strip()

    if not name:   # пустая строка — выйти
        break

    ram = int(input(f"  RAM для {name} (MB): "))
    monitored.append((name, ram))
    total_ram += ram

print(f"\nДобавлено {len(monitored)} процессов")
print(f"Суммарная RAM: {total_ram} MB")
for name, ram in monitored:
    print(f"  {name}: {ram} MB")
```

## 📖 Шаг 8 — while / else

`else` после `while` выполняется когда цикл завершился **нормально** (условие стало False), но НЕ при `break`:

```python
processes = ["chrome.exe", "discord.exe", "steam.exe", "python.exe"]
target    = "malware.exe"
idx       = 0

while idx < len(processes):
    if processes[idx] == target:
        print(f"Угроза найдена: {target}")
        break
    idx += 1
else:
    print(f"Процесс '{target}' не найден — система чиста")

# → Процесс 'malware.exe' не найден — система чиста
```

💡 while/else — редкий но удобный паттерн
> Используй когда нужно что-то сделать только если цикл прошёл полностью без прерывания.

## Задание

PC Booster запускает мониторинг в реальном времени. Напиши программу с текстовым меню.

**Дано:**
- Программа показывает меню с 4 пунктами и ждёт выбор пользователя
- Повторяет меню пока пользователь не выберет "Выход"

**Нужно:**
1. Показать меню: 1) Диагностика 2) Очистить Temp 3) Статистика 0) Выход
2. Запрашивать выбор в цикле `while True`
3. Считать количество выполненных действий
4. При выходе — вывести итог

**Ожидаемый вывод:**
```
╔══════════════════════════╗
║     PC Booster v1.0      ║
╠══════════════════════════╣
║  1. Диагностика          ║
║  2. Очистить Temp        ║
║  3. Статистика           ║
║  0. Выход                ║
╚══════════════════════════╝
Выбор: 1
→ Запускаю диагностику...
Выбор: 0
Сессия завершена. Действий: 1
```

💡 Подсказка 1
> Используй `while True:` и `break` при выборе `"0"`.

💡 Подсказка 2
> Счётчик действий: `actions += 1` после каждого пункта кроме "Статистика" и "Выход".

💡 Подсказка 3
> Для `else` (неверный выбор) просто выведи `"Неверный выбор"` и продолжи цикл.

## Решение

```python
actions = 0

while True:
    print()
    print("╔══════════════════════════╗")
    print("║     PC Booster v1.0      ║")
    print("╠══════════════════════════╣")
    print("║  1. Диагностика          ║")
    print("║  2. Очистить Temp        ║")
    print("║  3. Статистика           ║")
    print("║  0. Выход                ║")
    print("╚══════════════════════════╝")

    choice = input("Выбор: ").strip()

    if choice == "1":
        print("→ Запускаю диагностику...")
        actions += 1
    elif choice == "2":
        print("→ Очищаю папку Temp...")
        actions += 1
    elif choice == "3":
        print(f"→ Выполнено действий: {actions}")
    elif choice == "0":
        print(f"Сессия завершена. Действий: {actions}")
        break
    else:
        print("Неверный выбор. Введи число от 0 до 3.")
```

**Разбор:**
1. `while True:` → бесконечный цикл, выход только через `break`
2. `choice = input(...).strip()` → `.strip()` убирает случайные пробелы
3. Каждый `elif` обрабатывает свой пункт меню
4. `actions += 1` → увеличиваем счётчик только для реальных действий
5. `break` при `"0"` → единственный способ выйти из цикла
6. `else` → ловит любой неверный ввод и просит повторить

## ❌ Частые ошибки

🐛 Бесконечный цикл — забыл изменить переменную
> ```python
> count = 0
> while count < 5:
>     print(count)
>     # забыл count += 1 → цикл никогда не завершится!
>
> # ✓ Правильно
> while count < 5:
>     print(count)
>     count += 1
> ```

🐛 continue в while — инкремент должен быть ДО continue
> ```python
> i = 0
> while i < 5:
>     if i == 3:
>         continue      # ❌ i никогда не вырастет до 4 → бесконечный цикл!
>     print(i)
>     i += 1
>
> # ✓ Правильно
> while i < 5:
>     i += 1            # сначала инкремент
>     if i == 3:
>         continue
>     print(i)
> ```

🐛 Условие никогда не станет False
> ```python
> temp = 95
> while temp > 90:
>     print("Горячо!")
>     # temp не изменяется → вечный цикл!
> ```

## 💡 Заметки на полях

💡 while True + break — стандартный паттерн для меню
> ```python
> while True:
>     choice = input("Выбор: ")
>     if choice == "0":
>         break
>     # обработать choice
> ```
> Это самый читаемый способ сделать текстовое меню.

💡 Ctrl+C — экстренная остановка
> Если программа зависла в бесконечном цикле — нажми `Ctrl+C` в терминале. Python выбросит `KeyboardInterrupt` и остановится.

📝 Связь с PC Booster
> Главный цикл любого приложения — это `while True`. Программа запустилась → показывает меню → ждёт команды → выполняет → снова меню. Это и есть основа PC Booster: `while True: show_menu(); handle_command()`.
