## Главная идея

Словарь хранит пары **ключ → значение**. Вместо индекса `[0]` обращаешься по имени `["ram"]`. Идеально для хранения структурированных данных о чём-либо.

────────────────────
## Создание и доступ

```python
system = {
    "name":    "MyPC",
    "ram_gb":  16,
    "cpu":     "Intel i7",
    "temp":    72.5,
    "boosted": False
}

print(system["name"])          # → MyPC
print(system["ram_gb"])        # → 16
print(system.get("gpu", "N/A")) # → N/A (безопасный доступ с дефолтом)
```

⚠️ `dict[key]` vs `dict.get(key)`
> `system["gpu"]` — выбросит `KeyError` если ключа нет.
> `system.get("gpu")` — вернёт `None` если ключа нет.
> `system.get("gpu", "N/A")` — вернёт `"N/A"` если ключа нет.

────────────────────
## Методы словаря

Метод • Что делает
`d[key] = val` • Добавить или изменить
`del d[key]` • Удалить ключ
`d.keys()` • Все ключи
`d.values()` • Все значения
`d.items()` • Все пары (ключ, значение)
`d.get(key, default)` • Безопасный доступ
`d.update(d2)` • Обновить из другого словаря
`d.pop(key)` • Удалить и вернуть значение
`key in d` • Проверить наличие ключа
`len(d)` • Количество ключей

```python
# Итерация
for key, value in system.items():
    print(f"{key}: {value}")
```

processes = {

## Dict comprehension

```python
names = ["chrome", "discord", "steam"]
mems  = [850, 220, 340]

proc_dict = {name: mem for name, mem in zip(names, mems)}
print(proc_dict)  # → {'chrome': 850, 'discord': 220, 'steam': 340}

# С фильтром
heavy = {k: v for k, v in proc_dict.items() if v > 300}
print(heavy)  # → {'chrome': 850, 'steam': 340}
```

## Задание

PC Booster хранит данные о процессах в словаре. Создай базу данных и найди кандидатов на завершение.

**Дано:**
```python
processes = {
    "chrome.exe":    {"ram": 850, "cpu": 12.5, "safe": True},
    "svchost.exe":   {"ram":  45, "cpu":  0.5, "safe": False},
    "teams.exe":     {"ram": 680, "cpu":  9.1, "safe": True},
    "discord.exe":   {"ram": 220, "cpu":  3.2, "safe": True},
    "steam.exe":     {"ram": 340, "cpu":  7.8, "safe": True},
}
```

**Нужно:**
1. Вывести только процессы где `safe = True` и `ram > 200`
2. Посчитать суммарную RAM всех процессов
3. Найти самый тяжёлый процесс через `max()`

**Ожидаемый вывод:**
```
Кандидаты на завершение:
  chrome.exe  — RAM: 850 MB, CPU: 12.5%
  teams.exe   — RAM: 680 MB, CPU:  9.1%
  steam.exe   — RAM: 340 MB, CPU:  7.8%

Суммарная RAM: 2135 MB
Самый тяжёлый: chrome.exe (850 MB)
```

💡 Подсказка 1
> Итерируй по словарю: `for name, info in processes.items():` — `name` это ключ, `info` это вложенный словарь.

💡 Подсказка 2
> Доступ к значению: `info["ram"]`, `info["safe"]`. Проверка: `if info["safe"] and info["ram"] > 200`.

💡 Подсказка 3
> Для max: `max(processes, key=lambda k: processes[k]["ram"])` — возвращает ключ с максимальным RAM.

## Решение

```python
processes = {
    "chrome.exe":    {"ram": 850, "cpu": 12.5, "safe": True},
    "svchost.exe":   {"ram":  45, "cpu":  0.5, "safe": False},
    "teams.exe":     {"ram": 680, "cpu":  9.1, "safe": True},
    "discord.exe":   {"ram": 220, "cpu":  3.2, "safe": True},
    "steam.exe":     {"ram": 340, "cpu":  7.8, "safe": True},
}

print("Кандидаты на завершение:")
for name, info in processes.items():
    if info["safe"] and info["ram"] > 200:
        print(f"  {name:<15} RAM: {info['ram']:>3} MB, CPU: {info['cpu']:>4.1f}%")

total = sum(info["ram"] for info in processes.values())
print(f"\nСуммарная RAM: {total} MB")

heaviest = max(processes, key=lambda k: processes[k]["ram"])
print(f"Самый тяжёлый: {heaviest} ({processes[heaviest]['ram']} MB)")
```

**Разбор:**
1. `processes.items()` → возвращает пары `(ключ, значение)` → `("chrome.exe", {"ram": 850, ...})`
2. `info["safe"] and info["ram"] > 200` → два условия через `and`
3. `sum(info["ram"] for info in processes.values())` → generator expression считает сумму RAM
4. `max(processes, key=...)` → `key` говорит Python **по чему** сравнивать. Lambda берёт RAM из вложенного словаря
5. `heaviest` — это строка-ключ, а не весь словарь. Доступ: `processes[heaviest]["ram"]`
