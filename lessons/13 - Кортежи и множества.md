## Главная идея

**Кортеж** (`tuple`) — как список, но неизменяемый. **Множество** (`set`) — коллекция уникальных элементов без порядка. Оба решают конкретные задачи лучше списка.

────────────────────
## Кортеж (tuple)

```python
# Создание
point   = (10, 20)
process = ("chrome.exe", 850, 12.5)
single  = (42,)          # запятая обязательна для кортежа из 1 элемента

# Доступ — как у списка
print(process[0])    # → chrome.exe
print(process[-1])   # → 12.5

# Распаковка — очень удобно
name, mem, cpu = process
print(name)   # → chrome.exe
print(mem)    # → 850

# Обмен переменных без temp
a, b = 1, 2
a, b = b, a
print(a, b)   # → 2 1
```

### Когда использовать tuple

Ситуация • Почему tuple
Координаты, размеры • Не должны меняться
Возврат нескольких значений из функции • Стандартная практика
Ключ словаря • Список нельзя использовать как ключ
Константные конфиги • Защита от случайного изменения

```python
# Кортеж как ключ словаря
screen_res = {
    (1920, 1080): "Full HD",
    (2560, 1440): "2K",
    (3840, 2160): "4K",
}
print(screen_res[(1920, 1080)])  # → Full HD
```

## Множество (set)

```python
# Создание
s = {1, 2, 3, 3, 2}
print(s)          # → {1, 2, 3}  — дубликаты удалены

s2 = set([1, 2, 2, 3])   # из списка
empty = set()             # пустое (не {}, это словарь!)
```

### Операции с множествами

```python
a = {"chrome", "discord", "steam"}
b = {"steam", "teams", "discord"}

print(a & b)   # → {'discord', 'steam'}     — пересечение (есть в обоих)
print(a | b)   # → {'chrome', 'discord', 'steam', 'teams'}  — объединение
print(a - b)   # → {'chrome'}               — только в a
print(a ^ b)   # → {'chrome', 'teams'}      — только в одном из двух
```

### Методы множества

Метод • Что делает
`.add(x)` • Добавить элемент
`.remove(x)` • Удалить (ошибка если нет)
`.discard(x)` • Удалить (без ошибки если нет)
`.pop()` • Удалить случайный элемент
`x in s` • Проверить наличие — очень быстро

## Практика — PC Booster

```python
# Управление автозагрузкой через множества
current_startup  = {"OneDrive", "Teams", "Spotify", "Discord", "Steam", "Chrome"}
recommended_keep = {"Teams", "Discord", "Chrome"}

# Что можно отключить
to_disable = current_startup - recommended_keep
print(f"Отключить: {to_disable}")
# → Отключить: {'OneDrive', 'Spotify', 'Steam'}

# Что из рекомендованных уже есть
already_kept = current_startup & recommended_keep
print(f"Оставлено: {already_kept}")

# Конфиг разрешений — кортеж, не меняется
SYSTEM_PROCESSES = ("svchost.exe", "lsass.exe", "csrss.exe", "winlogon.exe")

def is_system(process_name):
    return process_name in SYSTEM_PROCESSES

print(is_system("svchost.exe"))   # → True
print(is_system("chrome.exe"))    # → False
```

## Задание

PC Booster анализирует лог с дублями. Используй множества для дедупликации.

**Дано:**
```python
log = [
    "chrome.exe", "svchost.exe", "chrome.exe",
    "discord.exe", "steam.exe", "discord.exe",
    "chrome.exe", "teams.exe"
]
system_procs = ("svchost.exe", "lsass.exe", "csrss.exe")
```

**Нужно:**
1. Убрать дубли из лога через `set()`
2. Отделить системные от пользовательских
3. Вывести количество уникальных и оба множества

**Ожидаемый вывод:**
```
Уникальных процессов: 5
Системные:    {'svchost.exe'}
Пользователь: {'chrome.exe', 'discord.exe', 'steam.exe', 'teams.exe'}
```

💡 Подсказка 1
> `unique = set(log)` — превращает список в множество, автоматически убирая дубли.

💡 Подсказка 2
> `system_procs` — кортеж (неизменяемый), потому что список системных процессов не должен меняться.

💡 Подсказка 3
> Для разделения используй операции множеств: `unique & set(system_procs)` — пересечение.

## Решение

```python
log = [
    "chrome.exe", "svchost.exe", "chrome.exe",
    "discord.exe", "steam.exe", "discord.exe",
    "chrome.exe", "teams.exe"
]
system_procs = ("svchost.exe", "lsass.exe", "csrss.exe")

unique = set(log)
sys_set = set(system_procs)

system_found = unique & sys_set
user_found   = unique - sys_set

print(f"Уникальных процессов: {len(unique)}")
print(f"Системные:    {system_found}")
print(f"Пользователь: {user_found}")
```

**Разбор:**
1. `set(log)` → убирает дубликаты: `{"chrome.exe", "svchost.exe", "discord.exe", "steam.exe", "teams.exe"}`
2. `system_procs` — кортеж `tuple`, неизменяемый. Список системных процессов константен.
3. `unique & sys_set` → **пересечение**: только те что есть в обоих множествах
4. `unique - sys_set` → **разность**: только те что в `unique`, но не в `sys_set`
5. `len(unique)` → количество уникальных элементов → `5`

## Заметки на полях

💡 Быстрое удаление дублей из списка
> ```python
> lst = [1, 2, 2, 3, 1]
> unique = list(set(lst))   # [1, 2, 3] — порядок не гарантирован
> ```

⚠️ Порядок в set не гарантирован
> ```python
> s = {"b", "a", "c"}
> print(s)   # может вывести {'a', 'b', 'c'} или по-другому
> ```
> Если нужны уникальные элементы С сохранением порядка — используй `dict.fromkeys(lst)`.

📝 Кортеж быстрее списка
> Для неизменяемых данных Python хранит кортежи эффективнее. В коде который работает с большими данными это заметно.
