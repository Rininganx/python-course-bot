## Главная идея

Область видимости определяет где переменная существует и откуда её можно прочитать. Переменная внутри функции — локальная, снаружи — глобальная. Путаница с этим — частый источник багов.

────────────────────
## LEGB — правило поиска имён

Python ищет переменную в таком порядке:

```
L — Local      (внутри текущей функции)
E — Enclosing  (во внешней функции, если есть)
G — Global     (на уровне модуля)
B — Built-in   (встроенные: print, len, range...)
```

```python
x = "global"

def outer():
    x = "enclosing"

    def inner():
        x = "local"
        print(x)   # → local  (нашёл в L)

    inner()
    print(x)       # → enclosing  (нашёл в E)

outer()
print(x)           # → global  (нашёл в G)
```

## global — изменить глобальную переменную

```python
boost_active = False

def enable_boost():
    global boost_active      # говорим Python: работаем с глобальной
    boost_active = True
    print("Буст включён")

enable_boost()
print(boost_active)   # → True
```

⚠️ `global` — используй редко
> Глобальные переменные делают код трудным для отладки. Лучший подход — передавать данные через аргументы и возвращать через `return`. `global` допустим для настроек приложения и флагов состояния.

────────────────────
## nonlocal — изменить переменную внешней функции

```python
def make_counter():
    count = 0

    def increment():
        nonlocal count
        count += 1
        return count

    return increment

counter = make_counter()
print(counter())   # → 1
print(counter())   # → 2
print(counter())   # → 3
```

## Практика — PC Booster

```python
# Конфигурация приложения — глобальные константы
APP_NAME    = "PC Booster"
VERSION     = "1.0.0"
MAX_TEMP    = 90
MAX_RAM_PCT = 85

# Состояние приложения
_boost_active    = False
_current_profile = "normal"
_total_optimizations = 0

def activate_boost():
    global _boost_active, _total_optimizations
    if _boost_active:
        print("Буст уже активен")
        return
    _boost_active = True
    _total_optimizations += 1
    print(f"[{APP_NAME}] Буст активирован (оптимизация #{_total_optimizations})")

def set_profile(profile: str):
    global _current_profile
    allowed = ("normal", "game", "silent")
    if profile not in allowed:
        print(f"Неизвестный профиль. Доступны: {allowed}")
        return
    _current_profile = profile
    print(f"Профиль установлен: {profile}")

def get_status() -> dict:
    return {
        "boost":   _boost_active,
        "profile": _current_profile,
        "total":   _total_optimizations,
    }

set_profile("game")
activate_boost()
activate_boost()   # второй вызов — уже активен
print(get_status())
```

Вывод:
```
Профиль установлен: game
[PC Booster] Буст активирован (оптимизация #1)
Буст уже активен
{'boost': True, 'profile': 'game', 'total': 1}
```

## Задание

PC Booster считает количество оптимизаций. Напиши систему из трёх функций с общим состоянием.

**Дано:**
- Глобальный счётчик `_clean_count = 0`
- `clean_temp()` — увеличивает счётчик, возвращает `"Temp очищен"`
- `clean_ram()` — увеличивает счётчик, возвращает `"RAM очищена"`
- `get_stats()` — возвращает строку `"Выполнено очисток: N"`

**Ожидаемый вывод:**
```
Temp очищен
RAM очищена
Temp очищен
Выполнено очисток: 3
```

💡 Подсказка 1
> Каждая функция начинается с `global _clean_count` — иначе Python создаст локальную переменную.

💡 Подсказка 2
> `_clean_count += 1` увеличивает глобальный счётчик. `return` возвращает строку-результат.

💡 Подсказка 3
> `get_stats()` только читает, не меняет — поэтому `global` не обязателен (но можно для единообразия).

## Решение

```python
_clean_count = 0

def clean_temp() -> str:
    global _clean_count
    _clean_count += 1
    return "Temp очищен"

def clean_ram() -> str:
    global _clean_count
    _clean_count += 1
    return "RAM очищена"

def get_stats() -> str:
    return f"Выполнено очисток: {_clean_count}"

print(clean_temp())
print(clean_ram())
print(clean_temp())
print(get_stats())
```

**Разбор:**
1. `_clean_count = 0` — глобальная переменная, живёт на уровне модуля
2. `global _clean_count` — без этой строки `_clean_count += 1` создаст **новую локальную** переменную
3. Каждая функция увеличивает счётчик и возвращает строку — делает две вещи
4. `get_stats()` только читает `_clean_count` — `global` не обязателен, но можно добавить для ясности
5. Счётчик накапливается между вызовами: 0 → 1 → 2 → 3

## Заметки на полях

💡 Константы пишут БОЛЬШИМИ_БУКВАМИ
> ```python
> MAX_TEMP = 90      # константа — не меняется
> current_temp = 72  # переменная — меняется
> ```
> Python не запрещает менять `MAX_TEMP`, но это соглашение: все понимают что трогать нельзя.

📝 Подчёркивание в начале = "приватное"
> ```python
> _internal_state = False   # _ говорит: это внутренняя переменная модуля
> ```
> Опять же — соглашение, не ограничение.
