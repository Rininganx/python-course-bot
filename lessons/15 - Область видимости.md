# Область видимости

> Где живёт переменная и откуда её можно прочитать.

## Theory

Python ищет переменные по порядку LEGB: Local → Enclosing → Global → Built-in. Внутри функции — локальные, снаружи — глобальные. `global` — изменить глобальную, `nonlocal` — изменить переменную внешней функции.

## Code

```python
# LEGB
x = "global"

def outer():
    x = "enclosing"
    def inner():
        x = "local"
        print(x)   # → local
    inner()
    print(x)       # → enclosing

outer()
print(x)           # → global

# Локальная переменная
def calculate(temp):
    result = temp * 1.8 + 32
    return result

calculate(100)
# print(result)  # → NameError!

# global — изменить глобальную
boost_active = False

def enable_boost():
    global boost_active
    boost_active = True

enable_boost()
print(boost_active)  # → True

# nonlocal — изменить переменную внешней функции
def make_counter():
    count = 0
    def increment():
        nonlocal count
        count += 1
        return count
    return increment

counter = make_counter()
print(counter())  # → 1
print(counter())  # → 2
```

## Practice

1. Напиши функцию с замыканием: `make_adder(n)` возвращает функцию, которая прибавляет `n`
2. Используй `nonlocal` для счётчика вызовов функции

## Answers

```python
# 1
def make_adder(n):
    def adder(x):
        return x + n
    return adder

add5 = make_adder(5)
print(add5(10))  # → 15

# 2
def make_calls_counter():
    count = 0
    def call():
        nonlocal count
        count += 1
        return f"Вызов #{count}"
    return call

f = make_calls_counter()
print(f())  # → Вызов #1
print(f())  # → Вызов #2
```

## Tips

- `global` используй редко — лучше передавать данные через аргументы и `return`
- Константы пиши БОЛЬШИМИ_БУКВАМИ: `MAX_TEMP = 90`
- Подчёркивание `_var` — соглашение о "приватности", не ограничение
