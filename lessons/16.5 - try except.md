## Главная идея

Программа падает когда встречает ошибку. `try/except` позволяет **перехватывать** ошибки и обрабатывать их gracefully — программа продолжает работать вместо краша.

────────────────────
## Базовый синтаксис

```python
try:
    # код который может упасть
    result = 10 / 0
except ZeroDivisionError:
    print("Нельзя делить на ноль!")
```

```
Нельзя делить на ноль!
```

────────────────────
## Ловим конкретные ошибки

```python
try:
    number = int(input("Введи число: "))
    result = 100 / number
except ValueError:
    print("Это не число!")
except ZeroDivisionError:
    print("Нельзя делить на ноль!")
```

────────────────────
## Получаем информацию об ошибке

```python
try:
    int("привет")
except ValueError as e:
    print(f"Ошибка: {e}")
```

```
Ошибка: invalid literal for int() with base 10: 'привет'
```

────────────────────
## except без указания типа (ловит всё)

```python
try:
    # опасный код
    risky_operation()
except Exception as e:
    print(f"Что-то пошло не так: {e}")
```

⚠️ > Ловить `Exception` — плохо. Лучше указывать конкретные типы. Иначе можно пропустить баг.

────────────────────
## else — если ошибок не было

```python
try:
    number = int(input("Число: "))
except ValueError:
    print("Не число!")
else:
    print(f"Ты ввёл: {number}")  # выполнится только если try прошёл
```

────────────────────
## finally — выполняется всегда

```python
try:
    file = open("data.txt")
    content = file.read()
except FileNotFoundError:
    print("Файл не найден")
finally:
    print("Этот блок выполнится ВСЕГДА")
```

`finally` используется для очистки — закрытие файлов, соединений, освобождение ресурсов.

────────────────────
## raise — бросаем свои ошибки

```python
def check_age(age):
    if age < 0:
        raise ValueError("Возраст не может быть отрицательным")
    if age < 18:
        raise PermissionError("Только для 18+")
    return True

try:
    check_age(15)
except PermissionError as e:
    print(f"Доступ запрещён: {e}")
```

```
Доступ запрещён: Только для 18+
```

────────────────────
## Практический пример: безопасный ввод числа

```python
def get_number(prompt="Введи число: "):
    while True:
        try:
            return float(input(prompt))
        except ValueError:
            print("Это не число, попробуй снова!")

num = get_number()
print(f"Ты ввёл: {num}")
```

────────────────────
## Практический пример: работа с API

```python
import requests

try:
    response = requests.get("https://api.example.com/data", timeout=5)
    response.raise_for_status()  # бросает ошибку если статус не 200
    data = response.json()
except requests.ConnectionError:
    print("Нет интернета")
except requests.Timeout:
    print("Сервер не отвечает")
except requests.HTTPError as e:
    print(f"Ошибка сервера: {e}")
else:
    print(f"Получено: {len(data)} элементов")
```

## Задание

Напиши функцию `safe_divide(a, b)` которая:
1. Пытается разделить a на b
2. Если b = 0 — возвращает `None` и печатает "Деление на ноль"
3. Если a или b не числа — возвращает `None` и печатает "Неверные аргументы"
4. В остальных случаях возвращает результат деления

- Используй `try/except` с `ZeroDivisionError` и `TypeError`
- `TypeError` возникает когда нельзя выполнить операцию с данными типами

────────────────────
## Решение

```python
def safe_divide(a, b):
    try:
        result = a / b
    except ZeroDivisionError:
        print("Деление на ноль")
        return None
    except TypeError:
        print("Неверные аргументы")
        return None
    else:
        return result

# Тестируем
print(safe_divide(10, 3))      # 3.333...
print(safe_divide(10, 0))      # None + "Деление на ноль"
print(safe_divide("10", 3))    # None + "Неверные аргументы"
print(safe_divide(10, "abc"))  # None + "Неверные аргументы"
```

**Разбор:**
- `try` — пытаемся выполнить деление
- `ZeroDivisionError` — ловим деление на ноль
- `TypeError` — ловим несовместимые типы
- `else` — возвращаем результат если всё ок
- Функция **никогда не падает** — всегда возвращает результат или `None`
