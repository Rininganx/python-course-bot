# Обработка ошибок (try/except)

> Программа продолжает работать вместо краша.

## Theory

`try/except` перехватывает ошибки. `try` — код который может упасть, `except` — что делать если упало. `else` — выполняется если ошибок не было. `finally` — выполняется всегда. `raise` — бросить свою ошибку.

## Code

```python
# Базовый синтаксис
try:
    result = 10 / 0
except ZeroDivisionError:
    print("Нельзя делить на ноль!")

# Ловим конкретные ошибки
try:
    number = int(input("Введи число: "))
    result = 100 / number
except ValueError:
    print("Это не число!")
except ZeroDivisionError:
    print("Деление на ноль!")

# Получаем информацию об ошибке
try:
    int("привет")
except ValueError as e:
    print(f"Ошибка: {e}")

# else — если ошибок не было
try:
    number = int(input("Число: "))
except ValueError:
    print("Не число!")
else:
    print(f"Ты ввёл: {number}")

# finally — выполняется всегда
try:
    file = open("data.txt")
    content = file.read()
except FileNotFoundError:
    print("Файл не найден")
finally:
    print("Выполнится ВСЕГДА")

# raise — бросаем свои ошибки
def check_age(age):
    if age < 0:
        raise ValueError("Возраст не может быть отрицательным")
    if age < 18:
        raise PermissionError("Только для 18+")
    return True
```

## Practice

1. Напиши функцию `safe_divide(a, b)` — деление с обработкой ошибок
2. Напиши функцию `safe_input(prompt)` — бесконечно запрашивает число до корректного ввода

## Answers

```python
# 1
def safe_divide(a, b):
    try:
        return a / b
    except ZeroDivisionError:
        print("Деление на ноль")
        return None
    except TypeError:
        print("Неверные аргументы")
        return None

print(safe_divide(10, 3))      # → 3.333...
print(safe_divide(10, 0))      # → None + сообщение

# 2
def safe_input(prompt="Введи число: "):
    while True:
        try:
            return float(input(prompt))
        except ValueError:
            print("Это не число, попробуй снова!")
```

## Tips

- Лови конкретные ошибки, не `Exception` — иначе пропустишь баги
- `finally` используется для очистки: закрытие файлов, соединений
- Функция с `try/except` **никогда не падает** — всегда возвращает результат или `None`
