# pip и виртуальные окружения

> Предыдущая тема: 20 - Реестр Windows
> Следующая тема: 21 - Процессы и память psutil

## Главная идея

Python — огромная экосистема. Тысячи библиотек решают типовые задачи: работа с сетью, GUI, базы данных, AI. `pip` — менеджер пакетов для установки этих библиотек. `venv` — изолированное окружение чтобы пакеты разных проектов не конфликтовали.

────────────────────
## pip — установка пакетов

```bash
# Установить пакет
pip install requests

# Установить конкретную версию
pip install requests==2.31.0

# Обновить пакет
pip install --upgrade requests

# Удалить пакет
pip uninstall requests

# Посмотреть все установленные пакеты
pip list

# Посмотреть информацию о пакете
pip show requests

# Экспортировать зависимости
pip freeze > requirements.txt

# Установить из файла
pip install -r requirements.txt
```

────────────────────
## Зачем venv?

Без venv все пакеты ставятся **глобально**. Проблемы:
- Проект A нужен `requests==2.28`, проект B — `requests==2.31`
- Глобальный Python засоряется ненужными пакетами
- На сервере может не быть прав на глобальную установку

**venv** создаёт **изолированную копию Python** для каждого проекта.

────────────────────
## Создание venv

```bash
# Создать окружение
python -m venv .venv

# Активировать (Windows)
.venv\Scripts\activate

# Активировать (Linux/Mac)
source .venv/bin/activate

# Деактивировать
deactivate
```

После активации в терминале появляется префикс `(.venv)`:

```
(.venv) C:\MyProject>
```

────────────────────
## Рабочий процесс

```bash
# 1. Создать проект
mkdir my_project
cd my_project

# 2. Создать venv
python -m venv .venv

# 3. Активировать
.venv\Scripts\activate

# 4. Установить пакеты
pip install requests psutil

# 5. Сохранить зависимости
pip freeze > requirements.txt

# 6. Работать!
python main.py
```

────────────────────
## requirements.txt

Файл со списком всех зависимостей проекта:

```txt
requests==2.31.0
psutil==5.9.5
customtkinter==5.2.0
```

```bash
# Установить все зависимости из файла
pip install -r requirements.txt
```

💡 > Всегда создавай `requirements.txt` для своих проектов. Это позволяет другим (и тебе через полгода) восстановить окружение одной командой.

## Практический пример: установка psutil

```bash
# Создаём проект
mkdir system_monitor
cd system_monitor
python -m venv .venv
.venv\Scripts\activate

# Устанавливаем psutil
pip install psutil

# Проверяем
pip list
```

```python
# main.py
import psutil

cpu = psutil.cpu_percent(interval=1)
ram = psutil.virtual_memory().percent
print(f"CPU: {cpu}%, RAM: {ram}%")
```

```bash
# Сохраняем зависимости
pip freeze > requirements.txt
```

## .gitignore

Папку `.venv` **нельзя** добавлять в Git — она слишком большая и специфична для твоей системы.

Создай файл `.gitignore`:

```gitignore
.venv/
__pycache__/
*.pyc
```

А `requirements.txt` **нужно** коммитить — по нему другие разработчики восстановят окружение.

## Задание

1. Создай папку `test_project`
2. Создай внутри venv
3. Активируй его
4. Установи пакет `rich` (красивый вывод)
5. Создай скрипт который выводит таблицу через `rich`
6. Сохрани зависимости в `requirements.txt`

- `python -m venv .venv` — создать окружение
- `.venv\Scripts\activate` — активировать
- `pip install rich` — установить пакет
- `from rich.console import Console` — импорт

────────────────────
## Решение

```bash
mkdir test_project
cd test_project
python -m venv .venv
.venv\Scripts\activate
pip install rich
```

```python
# main.py
from rich.console import Console
from rich.table import Table

console = Console()

table = Table(title="Системная информация")
table.add_column("Параметр", style="cyan")
table.add_column("Значение", style="green")

table.add_row("Python", "3.11")
table.add_row("OS", "Windows 11")
table.add_row("Пакеты", "rich установлен!")

console.print(table)
```

```bash
pip freeze > requirements.txt
cat requirements.txt
# rich==13.7.0
# markdown-it-py==3.0.0
# mdurl==0.1.2
# pygments==2.17.2
```

**Разбор:**
- Создали изолированное окружение `.venv`
- Установили `rich` только для этого проекта
- `requirements.txt` содержит все зависимости (rich + его зависимости)
- Другой разработчик сделает `pip install -r requirements.txt` и получит то же окружение
