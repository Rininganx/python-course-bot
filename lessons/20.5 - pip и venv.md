# pip и виртуальные окружения

> `pip` устанавливает пакеты, `venv` изолирует проекты.

## Theory
`pip` — менеджер пакетов Python для установки библиотек. `venv` создаёт изолированное окружение для каждого проекта, чтобы избежать конфликтов версий. `requirements.txt` хранит список зависимостей проекта.

## Code
```bash
# Установка пакетов
pip install requests
pip install requests==2.31.0
pip uninstall requests
pip list

# Создание виртуального окружения
python -m venv .venv
.venv\Scripts\activate  # Windows
deactivate

# Работа с зависимостями
pip freeze > requirements.txt
pip install -r requirements.txt
```

```python
# Пример использования пакета
import psutil
cpu = psutil.cpu_percent(interval=1)
print(f"Загрузка CPU: {cpu}%")
```

## Practice
1. Создай виртуальное окружение и установи пакет `rich`.
2. Напиши скрипт, который выводит таблицу через `rich`.
3. Сохрани зависимости в `requirements.txt`.

## Answers
```bash
# 1. Создание окружения
python -m venv .venv
.venv\Scripts\activate
pip install rich

# 3. Сохранение зависимостей
pip freeze > requirements.txt
```

```python
# 2. Скрипт с rich
from rich.console import Console
from rich.table import Table

console = Console()
table = Table(title="Система")
table.add_column("Параметр", style="cyan")
table.add_column("Значение", style="green")
table.add_row("Python", "3.11")
table.add_row("ОС", "Windows 11")
console.print(table)
```

## Tips
- Всегда создавай `requirements.txt` для своих проектов.
- Не добавляй `.venv/` в Git — она слишком большая.
- Используй `pip install package==version` для фиксации версий.
