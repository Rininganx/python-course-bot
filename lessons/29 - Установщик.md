## Главная идея

`.exe` из PyInstaller — это портативный файл. Установщик делает из него нормальное приложение: создаёт ярлык, добавляет в "Программы и компоненты", позволяет удалить через Панель управления.

────────────────────
## Inno Setup — проще всего

Скачай: https://jrsoftware.org/isinfo.php

```pascal
; PCBooster.iss — скрипт Inno Setup

[Setup]
AppName=PC Booster
AppVersion=1.0.0
AppPublisher=YourName
DefaultDirName={autopf}\PCBooster
DefaultGroupName=PC Booster
OutputBaseFilename=PCBooster_Setup_1.0.0
OutputDir=release
Compression=lzma2
SolidCompression=yes
PrivilegesRequired=admin

[Files]
Source: "dist\PCBooster.exe"; DestDir: "{app}"; Flags: ignoreversion
Source: "assets\*"; DestDir: "{app}\assets"; Flags: recursesubdirs

[Icons]
Name: "{group}\PC Booster"; Filename: "{app}\PCBooster.exe"
Name: "{commondesktop}\PC Booster"; Filename: "{app}\PCBooster.exe"

[Run]
Filename: "{app}\PCBooster.exe"; Description: "Запустить PC Booster"; Flags: postinstall nowait
```

Компиляция: открой `.iss` файл в Inno Setup → Build → Compile.

## NSIS — более гибкий вариант

Скачай: https://nsis.sourceforge.io

```nsis
; PCBooster.nsi

!define APP_NAME "PC Booster"
!define APP_VERSION "1.0.0"
!define EXE_NAME "PCBooster.exe"

Name "${APP_NAME}"
OutFile "PCBooster_Setup.exe"
InstallDir "$PROGRAMFILES\PCBooster"
RequestExecutionLevel admin

Section "Установка"
    SetOutPath "$INSTDIR"
    File "dist\PCBooster.exe"
    File /r "assets\"

    CreateShortcut "$DESKTOP\PC Booster.lnk" "$INSTDIR\PCBooster.exe"
    CreateShortcut "$SMPROGRAMS\PC Booster.lnk" "$INSTDIR\PCBooster.exe"

    WriteUninstaller "$INSTDIR\Uninstall.exe"

    WriteRegStr HKLM "Software\Microsoft\Windows\CurrentVersion\Uninstall\PCBooster" \
        "DisplayName" "${APP_NAME}"
    WriteRegStr HKLM "Software\Microsoft\Windows\CurrentVersion\Uninstall\PCBooster" \
        "UninstallString" "$INSTDIR\Uninstall.exe"
SectionEnd

Section "Удаление"
    Delete "$INSTDIR\PCBooster.exe"
    Delete "$DESKTOP\PC Booster.lnk"
    RMDir "$INSTDIR"
    DeleteRegKey HKLM "Software\Microsoft\Windows\CurrentVersion\Uninstall\PCBooster"
SectionEnd
```

## Задание

PC Booster распространяется через установщик. Напиши скрипт Inno Setup.

**Дано:**
- Готовый `PCBooster.exe` из PyInstaller
- Папка `assets/` с иконками

**Нужно:**
1. Создать `.iss` файл для Inno Setup
2. Настроить: имя, версия, папка установки
3. Добавить ярлыки на рабочий стол и в меню Пуск
4. Настроить деинсталляцию
5. Запустить компиляцию

**Ожидаемый результат:**
```
PCBooster_Setup_1.0.0.exe  — готовый установщик
```

При запуске установщика:
- Выбор папки установки
- Создание ярлыков
- Запись в "Программы и компоненты"
- Удаление через Панель управления

💡 Подсказка 1
> Скрипт `.iss` — это текстовый файл. Открой его в Inno Setup и нажми Build → Compile.

💡 Подсказка 2
> `{autopf}` — автоматически подставляет `C:\Program Files`. `{commondesktop}` — рабочий стол.

💡 Подсказка 3
> Для удаления: `WriteUninstaller "$INSTDIR\Uninstall.exe"` — создаёт `Uninstall.exe`.

## Решение

```
; === PCBooster.iss ===
; Открой в Inno Setup → Build → Compile

[Setup]
AppName=PC Booster
AppVersion=1.0.0
AppPublisher=YourName
DefaultDirName={autopf}\PCBooster
DefaultGroupName=PC Booster
OutputBaseFilename=PCBooster_Setup_1.0.0
OutputDir=release
Compression=lzma2
SolidCompression=yes
PrivilegesRequired=admin

[Files]
Source: "dist\PCBooster.exe"; DestDir: "{app}"; Flags: ignoreversion
Source: "assets\*"; DestDir: "{app}\assets"; Flags: recursesubdirs

[Icons]
Name: "{group}\PC Booster"; Filename: "{app}\PCBooster.exe"
Name: "{commondesktop}\PC Booster"; Filename: "{app}\PCBooster.exe"

[Run]
Filename: "{app}\PCBooster.exe"; Description: "Запустить PC Booster"; Flags: postinstall nowait

[UninstallDelete]
Type: filesandordirs; Name: "{app}\assets"
Type: filesandordirs; Name: "{app}\profiles"
```

**Разбор:**
1. `{autopf}` → `C:\Program Files` (автоматически для 64-бит)
2. `{commondesktop}` → общий рабочий стол всех пользователей
3. `{group}` → папка в меню Пуск
4. `Flags: recursesubdirs` → копирует все подпапки рекурсивно
5. `postinstall nowait` → запускает приложение после установки, не дожидаясь закрытия
6. `[UninstallDelete]` → удаление папок при деинсталляции (сами файлы удаляются автоматически)

💡 Inno Setup vs NSIS
> Inno Setup — проще, хватает для большинства проектов.
> NSIS — гибче, больше контроля, сложнее синтаксис.
> Начни с Inno Setup.

📝 Версионирование в имени файла
> `PCBooster_Setup_1.0.0.exe` — пользователь сразу видит версию.
> Обновляй версию в `.iss` при каждом релизе.
